## класс server
### атрибуты класса (общие)
 - VERSION: версия сервера (1.0.5) (эта версия будет у ВСЕХ созданных обьектов)
### атрибуты эеземпляра
- _name: Строка. Хранит очищенное от пробелов имя сервиса.
- _max_tasks: Целое число. Лимит задач, который прошел валидацию.
- _tasks: Список (list). Содержит названия текущих задач.
- _status: Объект ServiceStatus. Хранит текущее состояние (IDLE, WORKING и т.д.).

### Свойства (только чтение)
- name: Только для чтения. Возвращает имя сервиса.
- status: Только для чтения. Возвращает текстовое значение статуса (например, "idle").
- load_percentage: Только для чтения. Вычисляет текущий процент загрузки на лету.
- max_tasks: Доступно для чтения и изменения (через сеттер с проверкой).
- VERSION

### Сценарии работы кода
- проверка валидации (проверк входных двнных)
- Превышение лимита задач (добавление задачи overcharge сверх лимита задач с последующей остановокй работы сервера в режиме error)
- перевод в режим ожидания (перевод сервера в режим ожидания путем очистки списк задач)
- перевод в режим обслуживания (перевод сервера в режим обслуживания используя метод set_maintenance())
- изменение лимита задач через setter
