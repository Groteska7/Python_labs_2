# Наследование классов и интерфейсы (ЛР №4)

## Цель работы
Изучить интерфейсов через абстрактные базовые классы (ABC), научиться применять множественное наследование и использовать полиморфизм.

---

### Дочерние классы

#### 1. Вычислительный сервер **ComputeServer**
- **Дополнительные атрибуты**: `_cpu_power` (мощность), `_predicted_time` (прогноз времени).
- **Дополнительные методы**: `get_estimated_time`, `info` (переопределен).
- **Отличия**: Специализируется только на вычислениях. Имеет скорость обработки задач. При переходе в режим обслуживания уведомляет об остановке вычислительных потоков.

#### 2. Дата-сервер **StorageServer**
- **Дополнительные атрибуты**: `_all_memory` (общий объем), `_use_memory` (занято).
- **Дополнительные методы**: `info` (переопределен), `sync` (реализация интерфейса `ICloudSync`).
- **Отличия**: Вместо мощности оперирует памятью. Реализует дополнительный интерфейс облачной синхронизации. Перед техобслуживанием выполняет резервное копирование данных.

#### 3. Прокси-сервер **ProxyServer**
- **Дополнительные атрибуты**: `_ping` (задержка), `_target_server` (целевой сервер).
- **Дополнительные методы**: `take_target`, `pull_task`, `info`.
- **Отличия**: Умеет переадресовывать задачи на другие серверы. При техобслуживании разрывает соединения с целевыми серверами.

---

## Реализация интерфейсов

В проекте реализовано 3 интерфейса для обеспечения полиморфизма:

1.  **Manageable** (Интерфейс управления):
    - Метод `set_maintenance()`: Обязывает каждый сервер иметь логику перехода в режим обслуживания.
2.  **Diagnosable** (Интерфейс отчетности):
    - Метод `get_detailed_report()`: Позволяет получать унифицированный отчет о состоянии узла.
3.  **CloudSync** (Интерфейс синхронизации):
    - Метод `sync()`: Реализован выборочно (например, в `StorageServer`) для демонстрации множественного наследования.

### Ключевые особенности реализации:
- **Интерфейс как тип**: Использование аннотаций типов `list[IDiagnosable]` в функциях.
- **Универсальная функция**: Создана функция `process_diagnostics`, которая обрабатывает список любых серверов, используя `isinstance()` для проверки доступных интерфейсов (например, вызывает `sync()` только если объект поддерживает `ICloudSync`).
- **Полиморфизм**: Разные классы по-разному реализуют один и тот же метод интерфейса (например, диагностика выдает данные о CPU для одного сервера и о памяти для другого).
